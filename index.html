Productos

FORINLP EDAN v1.0 ya está disponible para su descarga en nuestro repositorio en línea, accesible a través de este enlace:Repositorio FORINLP EDAN.

Este conjunto de datos se entrega en dos formatos versátiles: Comma Separated Values (CSV), que facilita su implementación en cualquier software estadístico, y PostgreSQL (SQL), ideal para aquellos que prefieren trabajar con el software de base de datos PostgreSQL de código abierto. Además, hemos integrado una opción adicional para acceder a los datos mediante Web Map Service (WMS), brindando a los usuarios la flexibilidad de incorporar fácilmente la información en sus aplicaciones de Sistemas de Información Geográfica (SIG) preferidas desde cualquier ubicación

¡Estamos emocionados de poner a disposición esta valiosa herramienta para la comunidad!

Backup BDG EDAN

Descargar el Respaldo de la Base de Datos:

  1. Accede al enlace proporcionado: Google Drive.
  2. Localiza el archivo de respaldo de la base de datos (bdgforinlp.backup) en la carpeta.
  3. Descarga el archivo a tu máquina local.

Restaurar el Respaldo en pgAdmin4:

  1. Abre pgAdmin4 y conéctate al servidor PostgreSQL.
  2. En el panel izquierdo, navega hasta "Servidores" >> "PostgreSQL" >> (tu servidor) >> "Bases de Datos".
  3. Haz clic con el botón derecho en "Bases de Datos" y selecciona "Nueva Base de Datos...".
  4. En la ventana emergente, proporciona un nombre para la nueva base de datos, por ejemplo, bdgforinlp, y haz clic en "Guardar".
  5. Haz clic con el botón derecho en la nueva base de datos y selecciona "Restaurar...".
  6. En la pestaña "General", selecciona el archivo de respaldo descargado (bdgforinlp.backup) como "Nombre de archivo".
  7. En la pestaña "Opciones", asegúrate de que el esquema y las opciones de restauración sean adecuados. Puedes mantener las configuraciones predeterminadas.
  8. Haz clic en "Restaurar".
  9. Espera a que se complete el proceso de restauración. Al finalizar, deberías ver un mensaje indicando que la restauración fue exitosa.
  10. Ahora, en el panel izquierdo, expande la base de datos recién creada (bdgforinlp) y explora el esquema "public" para acceder a tus tablas y datos restaurados.

Servicio de Mapa Web (OGC)

Abre QGIS:

  1. Inicia QGIS en tu máquina.
  2. En la barra de menú de QGIS, ve a Capa -> Añadir capa WMS....
  3. Haz clic en el botón "Nuevo" para añadir un nuevo servidor WMS.

Configurar el Servidor WMS:

  1. En la ventana emergente "Nuevo servidor WMS", proporciona un nombre para el servidor (puede ser cualquier nombre descriptivo, por ejemplo, "BDGforINLP").
  2. En la sección "URL", ingresa la URL del WMS que proporcionaste (http://localhost:8081/geoserver/bdgforinlp/wms).
  3. Haz clic en "Aceptar" para cerrar la ventana.

Seleccionar Capas del WMS:

  1. En la ventana "Añadir capa WMS", deberías ver la lista de capas disponibles en el servidor WMS que acabas de agregar.
  2. Selecciona las capas que deseas añadir a tu proyecto en QGIS.
  3. Configurar Opciones (si es necesario):
  4. Puedes ajustar opciones adicionales, como el formato de la imagen, el sistema de referencia espacial (SRS), y otros parámetros según tus necesidades.
  5. Haz clic en "Aceptar" para añadir las capas seleccionadas al proyecto QGIS.
  6. Las capas del WMS deberían aparecer en la ventana de trabajo de QGIS. Puedes interactuar con ellas y realizar análisis geoespaciales según sea necesario.

Sobre el interfaz

Esta interfaz web representa un paso significativo en la accesibilidad de datos geográficos relacionados con eventos adversos, proporcionando una plataforma conveniente y efectiva para la interacción con estos datos. Al poner esta herramienta a disposición de los usuarios, se fomenta la comprensión, el análisis y la toma de decisiones informadas en relación con los eventos adversos más recurrentes. Además, la combinación de tecnologías libres y la publicación en la nube hacen que esta solución sea sostenible y escalable, lo que potencialmente beneficia a una amplia gama de usuarios.
